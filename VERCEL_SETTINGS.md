# ⚙️ Критически важные настройки Vercel для исправления ошибки Next.js

## Проблема
Vercel пытается автоматически определить Next.js, даже если проект статический. Нужно **явно указать** в настройках проекта, что это не Next.js.

## Решение: Настройка в Vercel Dashboard

### Шаг 1: Откройте настройки проекта
1. Перейдите на https://vercel.com/popovdenisn-1952s-projects/cat-tetris-base
2. Нажмите на вкладку **"Settings"**
3. Перейдите в раздел **"General"**

### Шаг 2: Измените Framework Preset (КРИТИЧЕСКИ ВАЖНО!)
1. Найдите секцию **"Framework Preset"**
2. Измените значение с **"Auto-detect"** или **"Next.js"** на **"Other"**
3. Это явно укажет Vercel, что проект НЕ использует Next.js

### Шаг 3: Настройте Build & Development Settings
В той же секции **"General"** найдите **"Build & Development Settings"**:

- **Build Command**: Оставьте **ПУСТЫМ** (не заполняйте)
- **Output Directory**: Укажите `.` (точка) или оставьте пустым
- **Install Command**: Оставьте **ПУСТЫМ** или `npm install` (но не обязательно)
- **Root Directory**: Укажите `.` (точка) - корневая директория

### Шаг 4: Сохраните настройки
1. Прокрутите вниз и нажмите **"Save"**
2. После сохранения Vercel автоматически запустит новый деплой

### Шаг 5: Проверьте деплой
1. Перейдите на вкладку **"Deployments"**
2. Дождитесь завершения нового деплоя
3. Проверьте логи - ошибка Next.js больше не должна появляться

## Альтернативный способ: Пересоздание проекта

Если настройки не помогают:

1. **Удалите проект** в Vercel Dashboard:
   - Settings → General → Delete Project

2. **Создайте проект заново**:
   - Нажмите "Add New Project"
   - Выберите репозиторий `denis198/cat-tetris-base`
   - **ВАЖНО:** При настройке выберите:
     - Framework Preset: **"Other"** (НЕ "Auto-detect"!)
     - Root Directory: `.`
     - Build Command: (оставьте пустым)
     - Output Directory: `.`
   - Нажмите "Deploy"

## Что было исправлено в коде

1. ✅ `vercel.json` - добавлены явные настройки `framework: null`, `buildCommand: null`
2. ✅ `package.json` - убраны пустые dependencies, упрощена структура
3. ✅ Создан `.vercelignore` для исключения ненужных файлов

## После применения настроек

После того как вы измените Framework Preset на "Other" в Dashboard:

1. Загрузите обновленные файлы:
   ```bash
   git add .
   git commit -m "Fix: явная конфигурация для статического проекта"
   git push origin main
   ```

2. Или просто запустите ручной деплой в Dashboard - настройки применятся автоматически

## Проверка

После деплоя проверьте:
- ✅ Нет ошибки "No Next.js version detected"
- ✅ Деплой завершается успешно
- ✅ Игра доступна по URL проекта
